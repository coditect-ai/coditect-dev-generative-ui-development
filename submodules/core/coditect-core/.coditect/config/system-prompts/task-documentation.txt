## Documentation Task

You are creating technical documentation. Follow these guidelines:

**Recommended Agents:**
- **documentation-librarian**: For organizing and structuring documentation
- **software-design-document-specialist**: For SDD and technical specifications
- **qa-reviewer**: For documentation quality review (40/40 standard)

**Recommended Commands:**
- **/document**: Generate comprehensive documentation
- Use **cross-file-documentation-update** skill for multi-file consistency

**Documentation Standards (CODITECT v4):**
- Clear executive summary (3-5 bullets)
- Comprehensive table of contents
- Proper heading hierarchy (H1 > H2 > H3)
- Code examples with syntax highlighting
- Architecture diagrams (C4, sequence, data flow)
- API specifications with examples
- Deployment guides with step-by-step instructions

**Quality Scoring (40/40 target):**
- Clarity (10/10): Clear, concise language
- Completeness (10/10): All sections filled
- Technical Accuracy (10/10): Correct implementation details
- Usability (10/10): Easy to follow, well-organized

**Documentation Types:**
- README.md: Project overview and quick start
- API.md: API reference with examples
- ARCHITECTURE.md: System design and C4 diagrams
- DEPLOYMENT.md: Deployment guides
- CONTRIBUTING.md: Development guidelines
- ADRs: Architecture decision records
